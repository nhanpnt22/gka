SURVIVAL–MEANING–GAIN MOTIVATION LOOP (SMG Loop)
A Unified Closed-Loop Constrained Optimization Theory of Motivation

Author: Phạm Ngọc Thanh Nhân (Brian Pham)
Date: March 24, 2026

--------------------------------------------------
1. FUNDAMENTAL AXIOM

All motivated behavior emerges from a closed-loop constrained optimization process governed by three irreducible components:

- Survival (S): enforces system viability over time
- Meaning (M): constrains admissible value structures via identity
- Gain (G): selects actions by maximizing expected net value

These components are not independent objectives, but interacting constraints and operators within a unified dynamical system.

--------------------------------------------------
2. SYSTEM CLASS

The SMG Loop is a:

Partially Observable Constrained Markov Decision Process (PO-CMDP)
with endogenous value projection and adaptive constraint weights.

Key properties:
- Partial observability via belief P_t
- State-dependent action space Ω(S_t)
- Endogenous constraints (Survival, Meaning)
- Time-varying weights λ_S(t), λ_M(t)

State transition:
S_{t+1} ~ P(· | S_t, a_t)

--------------------------------------------------
3. SYSTEM DYNAMICS

Closed-loop evolution:

S_t → M_t → π_t → a_t → S_{t+1} → (update) → (P_{t+1}, M_{t+1}, λ_{t+1})

Where:
- S_t: viability state
- M_t: identity-constrained value structure
- π_t: policy induced by optimization

Causal chain:
Survival → Feasibility → Meaning → Value → Gain → Action → Transition → Survival

--------------------------------------------------
4. FORMAL FRAMEWORK

State & Action:
- S_t ∈ S
- a_t ∈ Ω(S_t), where Ω(S_t) ≠ ∅ and is compact

Belief:
- P_t(s): probability distribution over states

Value:
- Raw: V_raw(s, a_t)

Meaning-constrained:
V_M(a_t) = E_{s ~ P_t}[ Proj_{M_t}(V_raw(s, a_t)) ]

--------------------------------------------------
5. OBJECTIVE FUNCTION

J(a_t) =
    V_M(a_t)
    − C(a_t)
    − λ_S(t) · Risk_S(a_t)
    − λ_M(t) · Deviation_M(a_t)

Policy:
π_t(S_t, M_t) = argmax_{a ∈ Ω(S_t)} J(a_t)

Existence:
If Ω(S_t) is compact and J(a_t) is continuous ⇒ optimal action exists.

--------------------------------------------------
6. CORE COMPONENTS

6.1 Survival (S) — Feasibility Constraint

Survival Functional:
Ψ(S_t) = E [ Σ_{k=0}^∞ γ^k · P(Survival_{t+k} | a_{t:t+k}) ]

Risk:
Risk_S(a_t) = 1 − E[ P(Survival_{t+1} | a_t) ]

Role:
- Defines feasible set Ω(S_t)
- Constrains transitions and resource usage

--------------------------------------------------
6.2 Meaning (M) — Identity Constraint

Projection operator:
Proj_{M_t}: V_raw → V_M

Formal definition:
Proj_{M_t}(V) = argmin_{v ∈ ℳ_t} ||v − V||

Where:
ℳ_t ⊂ ℝ^n is the identity-consistent value manifold.

Properties:
- Idempotent
- Bounded
- Order-preserving

Deviation:
Deviation_M(a_t) = D(V_M(a_t), V_identity)

Role:
- Filters value space
- Enforces identity coherence

--------------------------------------------------
6.3 Gain (G) — Optimization Operator

G ≡ argmax_a J(a_t)

Role:
- Selects optimal action under constraints
- Produces policy π_t

--------------------------------------------------
7. ADAPTIVE CONSTRAINT DYNAMICS

λ_S(t) = f_S(risk_t, uncertainty_t)
λ_M(t) = f_M(identity pressure_t, context_t)

Properties:
- λ_S(t), λ_M(t) ≥ 0
- Bounded, time-varying, state-dependent

Constraints are endogenous and adaptive.

--------------------------------------------------
8. LEARNING AND UPDATE DYNAMICS

Belief update (Bayesian):
P_{t+1} = Bayes(P_t, a_t, S_{t+1})

Identity update:
M_{t+1} = g(M_t, S_{t+1})

Constraint adaptation:
λ_S(t+1) = f_S(observed risk)
λ_M(t+1) = f_M(observed deviation)

These updates complete the closed-loop adaptive system.

--------------------------------------------------
9. FUNDAMENTAL TRADE-OFFS

- Gain vs Survival: value ↑ ⇒ risk ↑
- Gain vs Meaning: efficiency ↑ ⇒ identity violation
- Meaning vs Survival: identity ⇒ costly sacrifice

No system eliminates trade-offs — only balances them.

--------------------------------------------------
10. SYSTEM LAW

π_t = argmax_{a ∈ Ω(S_t)} J(a_t)

Subject to:
- Survival feasibility
- Meaning consistency

--------------------------------------------------
11. STABILITY CONDITIONS

System is stable iff:
- Ψ(S_t) ≥ S_min
- Deviation_M(a_t) ≤ ε_M
- 0 < λ_S(t), λ_M(t) < ∞
- Ω(S_t) ≠ ∅
- Policy is Bellman-consistent

--------------------------------------------------
12. FAILURE MODES

- Survival Collapse: Risk_S → 1
- Meaning Collapse: Deviation_M → ∞
- Gain Runaway: λ_S, λ_M → 0
- Constraint Lock: λ → ∞

--------------------------------------------------
13. CRITICAL BALANCE LAW

- Survival-dominant → stagnation
- Meaning-dominant → extremism
- Gain-dominant → collapse

Stability requires:
λ_S(t), λ_M(t) > 0 and dynamically balanced

--------------------------------------------------
14. MEASURABILITY

Survival:
- continuation probability
- risk exposure
- resilience

Meaning:
- value coherence
- entropy
- identity stability

Gain:
- expected utility
- reward accumulation
- efficiency

--------------------------------------------------
15. MASTER THEOREM

Motivation is not a force — it is an emergent policy of a system that:
- must survive
- must remain identity-coherent
- maximizes value under constraints

--------------------------------------------------
16. CANONICAL FORM

π_t = argmax_{a ∈ Ω(S_t)} [
    E_{s ~ P_t}[ Proj_{M_t}(V_raw(s, a)) ]
    − C(a)
    − λ_S(t) · (1 − P(Survival_{t+1} | a))
    − λ_M(t) · D(V_M(a), V_identity)
]

SURVIVAL–MEANING–GAIN OPERATIONAL MODEL (SMG-OM)
An Applied Decision System for Constrained Motivation Execution

Author: Phạm Ngọc Thanh Nhân (Brian Pham)
Date: March 24, 2026

--------------------------------------------------
1. PURPOSE

The SMG-OM operationalizes the SMG theory into a real-time sequential decision system.

Core Question:
“What is the optimal action given state, uncertainty, identity, and constraints?”

The system selects actions by jointly optimizing:
- Survival (viability)
- Meaning (identity coherence)
- Gain (value maximization)

--------------------------------------------------
2. SYSTEM STATE

At time t, the system is defined by:

- State: S_t ∈ S
- Belief: P_t(s), a probability distribution over S (partial observability)
- Identity Model: M_t ⊂ V, the identity-consistent value manifold
- Constraint Weights: λ_S(t), λ_M(t) ≥ 0
- Action Space: Ω(S_t)

State transition:
S_{t+1} ~ P(· | S_t, a_t)

--------------------------------------------------
3. SURVIVAL FEASIBILITY

Define survival-feasible action set:

Ω_S(S_t) = { a ∈ Ω(S_t) | Risk_S(a) ≤ τ_S(t) }

Where:
- Risk_S(a) = 1 − P(Survival_{t+1} | S_t, a)
- τ_S(t): maximum tolerable survival risk

Fallback rule:
If Ω_S(S_t) = ∅:
    a_t = argmin_{a ∈ Ω(S_t)} Risk_S(a)

All optimization is restricted to Ω_S(S_t) unless fallback is triggered.

--------------------------------------------------
4. VALUE CONSTRUCTION

Raw value:
V_raw(s, a)

Meaning-projected value:
V_M(a) = E_{s ~ P_t} [ Proj_{M_t}(V_raw(s, a)) ]

Projection operator:
Proj_{M_t}(V) = argmin_{v ∈ M_t} || v − V ||

Where:
- M_t is the identity-consistent value manifold
- ||·|| is a norm (e.g., L2 or information-theoretic metric)

Interpretation:
M_t encodes admissible values under identity constraints.

--------------------------------------------------
5. OBJECTIVE FUNCTION

Immediate objective:

J(a) =
    V_M(a)
    − C(a)
    − λ_S(t) · Risk_S(a)
    − λ_M(t) · Deviation_M(a)

Where:
- C(a): resource cost function
- V_identity ∈ M_t: reference identity-consistent value
- Deviation_M(a) = D(V_M(a), V_identity)
- D(·,·): divergence metric (e.g., L2, KL, entropy-based)

--------------------------------------------------
6. VALUE FUNCTION (BELLMAN FORM)

Define optimal value function:

V*(S_t) =
    max_{a ∈ Ω_S(S_t)} [
        J(a)
        + γ · E_{S_{t+1} ~ P(· | S_t, a)} [ V*(S_{t+1}) ]
    ]

Where:
- γ ∈ (0,1): temporal discount factor

--------------------------------------------------
7. POLICY

Define action-value function:

Q(S_t, a) =
    J(a)
    + γ · E_{S_{t+1} ~ P(· | S_t, a)} [ V*(S_{t+1}) ]

Optimal policy:

π_t(S_t, M_t) =
    argmax_{a ∈ Ω_S(S_t)} Q(S_t, a)

--------------------------------------------------
8. OPERATIONAL PIPELINE

Step 1 — Feasibility Filter:
    Construct Ω_S(S_t)

Step 2 — Value Projection:
    Compute V_M(a)

Step 3 — Penalty Computation:
    Compute Risk_S(a), Deviation_M(a)

Step 4 — Objective Evaluation:
    Compute J(a)

Step 5 — Optimization:
    Select optimal action a_t* via π_t

--------------------------------------------------
9. EXECUTION

Execute optimal action a_t*

Observe next state:
S_{t+1}

--------------------------------------------------
10. ADAPTIVE UPDATE LOOP

Belief Update (Bayesian):
P_{t+1} = Bayes(P_t, a_t*, S_{t+1})

Identity Update:
M_{t+1} = g(M_t, S_{t+1})

Constraint Adaptation:
λ_S(t+1) = f_S(observed survival risk, uncertainty)
λ_M(t+1) = f_M(observed identity deviation, context)

--------------------------------------------------
11. EXPLORATION EXTENSION (OPTIONAL)

Augmented objective:

J'(a) = J(a) + β · H(P_{t+1}(· | S_t, a))

Where:
- H(P) = − Σ_s P(s) log P(s)
- β ≥ 0: exploration coefficient

--------------------------------------------------
12. RUNTIME REGIMES

Survival-Dominant:
λ_S >> λ_M → conservative behavior

Meaning-Dominant:
λ_M >> λ_S → identity rigidity

Gain-Dominant:
λ_S, λ_M → 0 → unstable optimization

Balanced Regime:
λ_S, λ_M > 0 → adaptive optimal behavior

--------------------------------------------------
13. FAILURE HANDLING

If Risk_S(a) → 1:
    increase λ_S

If Deviation_M(a) is high:
    increase λ_M

If stagnation detected:
    decrease dominant constraint weight

If collapse risk detected:
    restrict Ω(S_t)

--------------------------------------------------
14. SYSTEM PROPERTIES

- Closed-loop adaptive system
- Constrained stochastic optimal control
- Identity-projected value space
- Time-consistent (Bellman recursion)
- Trade-off balancing (Survival–Meaning–Gain)
- Guaranteed action existence if Ω(S_t) is compact

--------------------------------------------------
15. OUTPUT

At each time step, the system produces:

- Optimal action: a_t*
- Value estimate: V*(S_t)
- Updated belief: P_{t+1}
- Updated identity: M_{t+1}
- Updated constraints: λ_S(t+1), λ_M(t+1)

--------------------------------------------------
16. INTERPRETATION

The SMG-OM is a general-purpose decision engine where:

- Survival defines feasibility
- Meaning defines admissibility
- Gain drives optimization

Motivation emerges as the policy of a constrained, identity-consistent, survival-bounded optimization process.

    D(v,v) = 0,   D ≥ 0

        − λ_S · R_S(a)

λ_S(t+1) =
 Π_[λ_min, λ_max] [
   λ_S(t) + α_t · f_S(Δ_S(t)) + ξ_t^S

   λ_M(t) + α_t · f_M(Δ_M(t)) + ξ_t^M
 ]

Where:
- α_t > 0, with Σ α_t = ∞, Σ α_t² < ∞
- f_S, f_M are Lipschitz, monotone increasing
- f_S(0) = 0, f_M(0) = 0
- ξ_t^S, ξ_t^M are bounded martingale difference sequences
- Π denotes projection onto admissible bounds

τ_S(t+1) =
 Π_[0,1] [
   τ_S(t) − β_t · h_S(Δ_S(t))

Where:
- β_t > 0, Σ β_t = ∞, Σ β_t² < ∞
- h_S is Lipschitz and monotone increasing
- h_S allows both tightening and relaxation depending on Δ_S(t)

Interpretation:
- High risk ⇒ tighten feasibility (lower τ_S)
- Low risk ⇒ relax feasibility (increase τ_S)

- d_t ≤ ε_M
- λ_min ≤ λ_S(t), λ_M(t) ≤ λ_max

𝒱 :=

  d ≤ ε_M,
  λ_min ≤ λ_S, λ_M ≤ λ_max

- Δ_M = 0
- λ_S*, λ_M* ∈ (λ_min, λ_max)

(A1) Step sizes:
 Σ α_t = ∞, Σ α_t² < ∞

(A2) Regularity:
 f_S, f_M, h_S are Lipschitz

(A3) Noise:
 martingale difference, bounded variance

(A4) Feasibility:
 A_b ≠ ∅

Then:

(λ_S(t), λ_M(t), τ_S(t))
converge almost surely to an internally chain transitive invariant set
of the associated limiting ODE.

If the ODE admits a globally asymptotically stable equilibrium:
⇒ convergence to (λ_S*, λ_M*, τ_S*)
